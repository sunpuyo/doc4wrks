# 부서 정보 관리 - (POST) /admin/departments/bulk

이 API 는 부서 정보를 한 번에 추가, 수정 또는 삭제하는 데 사용됩니다. 현재 회사 내의 모든 부서 정보를 아래
형식의 페이로드에 담아 전달하면, 웍스 AI 시스템은 다음과 같은 방식으로 처리합니다.

- 1. 추가: 페이로드에는 존재하지만 웍스 AI 시스템에 동일한 code 를 가진 부서가 없는 경우, 해당
- 부서를 새로 추가합니다.
- 2. 수정: 페이로드에 존재하고 웍스 AI 시스템에도 동일한 code 를 가진 부서가 있는 경우, 페이로드의
- 메타 정보( meta )와 웍스 AI 시스템의 메타 정보를 비교하여 다를 경우 웍스 AI 시스템의 부서 정보를
- 페이로드에 맞춰 수정합니다.
- 3. 삭제: 다음의 경우, 해당 부서를 삭제합니다. 단, 현재 사용자에게 할당된 부서는 삭제에 실패합니다.


- a. (기본) 페이로드에 존재하고 웍스 AI 시스템에도 동일한 code 를 가지면서 delete=true 인
- 경우,
- b. (implicitDeletion 파라메터가 true 인 경우) 페이로드에 존재하지 않지만, 웍스 AI 시스템에
- 존재하는 경우


# RateLimit: 이 API 는 하루 최대 100 번 호출 가능합니다.

# [Parameter]

implicitDeletion: (선택, default=false) false 인 경우 명시적으로 delete 여부를 지정한 부서만
삭제합니다. true 로 설정한 경우는 payload 에 포함되어 있지 않은 부서를 삭제합니다.

# [Payload]

id

○ code: (필수) 수정할 부서의 코드를 입력합니다.

- delete: 삭제 여부 (default: false)
- meta


- ○ name: (추가 시 필수, 수정 시 선택) 수정할 부서명을 입력합니다.
- ○ parentCode: (추가 시 필수, 수정 시 선택) 상위 부서를 변경할 경우 입력합니다. (빈 문자열 =
- 최상위 단계)
- ○ memo: (선택) 사용자가 임의의 정보를 기입해 사용가능한 메모 필드 입니다.

# 사용자 정보 관리 - (POST) /admin/users/bulk

이 API는 기존 사용자의 메타 정보를 일괄적으로 수정하거나 삭제하는 데 사용됩니다. 페이로드에 아래
형식으로 현재 사용자 정보를 모두 담아 전달하면 웍스 AI 시스템은 다음과 같이 처리합니다.

- 추가: 개인 정보 보호를 위해 API 를 통한 사용자 추가는 지원하지 않습니다. 페이로드에 존재하지만
- 웍스 AI 시스템에 동일한 ID 를 가진 사용자가 없는 경우, 별다른 조치를 취하지 않고 넘어갑니다. 이
- 경우 E1027, 사용자 정보가 존재하지 않습니다. 라는 에러가 발생하나 무시하셔도 됩니다.
- 수정: 페이로드에 존재하고 웍스 AI 시스템에도 동일한 ID 를 가진 사용자가 있는 경우, 페이로드의 메타
- 정보( meta )와 웍스 AI 시스템의 메타 정보를 비교하여 다를 경우 웍스 AI 시스템의 사용자 정보를
- 페이로드에 맞춰 수정합니다.
- 삭제: 다음의 경우, 해당 사용자를 탈퇴 처리합니다.


- ○ (기본) 페이로드에 존재하고 웍스 AI 시스템에도 동일한 ID(userld 또는 email)를 가지면서
- delete=true 인 경우,
- ○ (implicitDeletion 파라메터가 true 인 경우) 페이로드에 존재하지 않지만, 웍스 AI 시스템에
- 존재하는 경우 (주의! payload 작성 실수 등의로 의도치 않게 유저를 삭제할 수도 있으니
- 사용상의 주의가 필요합니다. 가급적이면 payload 의 delete 기능을 사용하기를 권장합니다)


RateLimit: 이 API 는 하루 최대 100 번 호출 가능합니다.

# [Parameter]

idType: 페이로드에 사용할 ID 형식을 지정합니다. userld 또는 email 을 사용할 수 있으며,
기본값은 userld 입니다. (idType, userld 는 대소문자를 구분합니다. 잘못된 정보 입력시 400 에러가
반환됩니다)

AI3

implicitDeletion: (선택, default=false) false 인 경우 명시적으로 delete 여부를 지정한 사용자만
삭제합니다. true 로 설정한 경우는 payload 에 포함되어 있지 않은 사용자를 삭제합니다.

# [Payload]

# id

- ○ userld: 사용자 Id (get admin/users 에서의 id)
- ○ email: 사용자 email


- delete: 삭제 여부 (default: false)
- meta


- ○ enabled: 대상의 사용 가능 여부를 활성화(true) 또는 비활성화(false) 합니다.
- ○ name: 기업 내에서 표시되는 유저의 별칭을 변경합니다.
- ○ departmentCode: 사용자의 부서를 설정합니다.
- ○ workspaceld: (DEPRECATED) 이동할 워크스페이스의 ID 를 입력합니다. 해당 정보가
- 필요하면 문의주시기 바랍니다.
